Roadmap do Projeto Crush4Pets

Progresso de Desenvolvimento & Planejamento

Data Atual: 22 de Abril de 2025

Visão Geral do Projeto

Este documento apresenta o roadmap de desenvolvimento para o ecossistema Crush4Pets, que consiste em múltiplos serviços integrados trabalhando juntos para fornecer uma solução abrangente de gestão para pets.

O desenvolvimento está estruturado em seis sprints semanais, com revisões regulares e sessões de planejamento para garantir a entrega pontual de funcionalidades e correções de bugs.

Cronograma de Sprints

Sprint 1: Configuração Inicial

31 de Março - 7 de Abril de 2025

Início: 31 de Março de 2025 | Revisão: 7 de Abril de 2025 (Hoje)

  • Criar uma empresa no GitHub, chamada crush-4-pets
  • Enviar todos os 4 projetos para o GitHub
  • Criar um projeto orquestrado para automatizar o processo de inicialização local
  • Identificar todas as arquiteturas publicadas no Digital Ocean
  • Implementar Digital Ocean no projeto de upload, removendo os buckets AWS S3
  • Identificar local de publicação ausente para sistema.crush4pets.com.br
  • Correção de vários bugs encontrados no projeto do ERP

Sprint 2: Investigação e Preparação

8 de Abril - 14 de Abril de 2025

Planejamento: 8 de Abril de 2025 | Revisão: 14 de Abril de 2025

  • Investigar onde o projeto sistema.crush4pets.com.br está publicado
  • Abrir chamado na AWS para auxiliar na localização dos recursos
  • Definir estratégia de trabalho para o sistema.crush4pets.com.br
  • Preparar ambiente para implementação das funcionalidades na próxima sprint

Nota: As tarefas originalmente planejadas para esta sprint foram transferidas para a Sprint 3 devido ao tempo necessário para investigação e preparação do ambiente.

Sprint 3: Infraestrutura e Implementação

15 de Abril - 21 de Abril de 2025

Planejamento: 15 de Abril de 2025 | Revisão: 21 de Abril de 2025

  • Criar endpoints de resetar a senha no projeto Path
  • Criar o processo de deploy via CI/CD do GitHub Actions
  • Identificar divergências entre o código fonte entregue e o publicado no projeto Path
  • Fazer backup do banco de dados
  • Encontrar o lugar onde é feito a gestão dos DNS das aplicações
  • Criar o CDN na DigitalOcean e criar o DNS para ele
  • Criar o e-mail suporte@crush4pets.com
  • Configurar o servidor de envio de e-mail pela aplicação
  • Realizar ajustes na aplicação React Native (fluxo de reset de senha)

Nota: Foi identificado que não será possível usar o deploy do GitHub Actions devido a múltiplos problemas. Serão necessários ajustes pontuais no servidor após restauração do backup. Também foi identificado que o upload de arquivos e reset de senha no app React Native dependiam de um aplicativo pessoal do desenvolvedor anterior.

Sprint 4: Implantação do projeto Sistema na Digital Ocean

22 de Abril - 28 de Abril de 2025

Planejamento: 22 de Abril de 2025 | Revisão: 28 de Abril de 2025

  • Criar novo Droplet
  • Configurar DNS
  • Corrigir problemas mencionados pelo cliente
  • Refazer apontamentos corretos

Sprint 5: Realizar testes e submeter para as lojas

29 de Abril - 5 de Maio de 2025

Planejamento: 29 de Abril de 2025 | Revisão: 5 de Maio de 2025

  • Restaurar backup e finalizar ajustes no servidor do projeto Path
  • Implementar endpoint para subir imagens na Digital Ocean
  • Testar os demais fluxos da aplicação React Native
  • Reimplementar a funcionalidade de upload no projeto Path ou ERP
  • Submeter todas as correções de bugs para as lojas
  • Submeter todas as funcionalidades implementadas para as lojas
  • Preparar documentação para as submissões nas lojas
  • Conduzir testes pré-submissão

Sprint 6: Implementação de Feedback das Lojas

6 de Maio - 12 de Maio de 2025

Planejamento: 6 de Maio de 2025 | Revisão: 12 de Maio de 2025

  • Resolver todos os feedbacks da revisão das lojas
  • Corrigir todas as solicitações retornadas pelas lojas
  • Reenviar aplicativos atualizados
  • Teste final de garantia de qualidade
